# B1完美图形匹配详解

本文档详细解释B1完美图形匹配的相似度计算逻辑。

---

## 概述

B1完美图形匹配是一种**形态相似度匹配算法**，用于将当前选出的股票与10个历史成功案例进行多维相似度比较，并按相似度排序输出。

**核心目标**：找到与历史成功案例在**形态结构**上相似的股票，而非价格绝对值相似的股票。

## 匹配维度与权重

B1完美图形匹配基于**四个维度**计算相似度：

| 维度 | 权重 | 说明 |
|------|------|------|
| **双线结构** | 30% | 知行短期趋势线与多空线的相对位置、斜率、发散程度 |
| **KDJ状态** | 20% | J值位置、金叉状态、趋势方向 |
| **量能特征** | 25% | 成交量变化模式（缩量后放量、持续放量等） |
| **价格形态** | 25% | 归一化价格曲线相似度（DTW算法） |

---

## 相似度计算详解

### 1. 双线结构相似度（30%）

**核心思想**：比较股票相对于知行趋势线的位置关系，而非绝对价格。

#### 特征提取

对于每个案例和候选股票，提取以下特征：

| 特征 | 计算方法 | 含义 |
|------|----------|------|
| `short_vs_bullbear` | 短期趋势 / 多空线 | 两条趋势线的相对位置 |
| `short_slope` | (近1日斜率 / 近5日斜率 - 1) × 100% | 短期趋势线变化率 |
| `price_vs_short_pct` | (收盘价 - 短期趋势) / 短期趋势 × 100% | 价格相对于短期趋势的偏离百分比 |
| `price_bias_pct` | (收盘价 - 双线平均) / 双线平均 × 100% | 双线乖离率 |
| `trend_spread_pct` | (短期趋势 - 多空线) / 多空线 × 100% | 趋势线发散程度 |
| `is_in_bowl` | 短期趋势 > 收盘价 > 多空线 | 是否在"碗"中 |

#### 相似度计算

```python
# 1. 短期/多空比值相似（允许±10%误差）
ratio_diff = abs(cand_short_vs_bullbear - case_short_vs_bullbear)
sim = max(0, 1 - ratio_diff / 0.10)

# 2. 斜率方向一致性（最重要）
if 同向:
    sim = max(0.7, 1 - slope_diff / 10)  # 同向至少0.7分
else:
    sim = max(0, 0.3 - slope_diff / 20)  # 反向大幅降低

# 3. 价格偏离相似（允许±10%差异）
price_bias_diff = abs(cand_price_bias - case_price_bias)
sim = max(0, 1 - price_bias_diff / 10)

# 4. 位置一致性
if 都在碗中或都在碗外:
    sim = 1.0
else:
    sim = 0.2  # 位置不同惩罚重
```

**为什么用百分比而非绝对值？**

假设两个形态完全相同的股票：
- 股票A：价格20元，短期趋势线20.5元
- 股票B：价格100元，短期趋势线102.5元

如果使用绝对值比较：
- 股票A偏离：20/20.5 = 0.975
- 股票B偏离：100/102.5 = 0.976
- 差异微小，相似度高 ✅

但如果使用之前的绝对差值：
- 股票A偏离：-0.5元
- 股票B偏离：-2.5元  
- 差异很大，相似度低 ❌

使用**百分比偏离**可以消除价格绝对值的影响，专注于形态比较。

### 2. KDJ状态相似度（20%）

**核心思想**：比较KDJ指标的相对状态，而非绝对数值。

#### 特征提取

| 特征 | 计算方法 | 含义 |
|------|----------|------|
| `j_value` | J值具体数值 | J值绝对值 |
| `j_position` | J值 ≤20为"低位"，≥80为"高位"，其他为"中位" | J值区域 |
| `k_cross_d` | 前一日K<D且当日K>D | 是否金叉 |
| `j_rebound` | J值近3日回升 | J值趋势 |

#### 相似度计算

```python
# 1. J值位置一致性
if 同为低位 or 同为中位 or 同为高位:
    sim = 1.0
elif 一个低位一个中位:
    sim = 0.8
else:
    sim = 0.4

# 2. J值数值相似（允许±30误差）
j_diff = abs(cand_j - case_j)
sim = max(0, 1 - j_diff / 30)

# 3. 金叉状态一致性
if 同为金叉或同非金叉:
    sim = 1.0
else:
    sim = 0.6
```

---

### 3. 量能特征相似度（25%）

**核心思想**：比较成交量变化模式，而非绝对成交量。

#### 特征提取

| 特征 | 计算方法 | 含义 |
|------|----------|------|
| `avg_volume_ratio` | 近10日均量 / 前10日均量 | 近期量能变化 |
| `volume_trend` | 线性回归分类 | 量能趋势分类 |
| `shrink_then_expand` | 前半月缩量且后半月放量 | 缩量后放量模式 |
| `max_volume_ratio` | 单日最大量比 | 最大放量倍数 |

#### 量能趋势分类

```python
if slope > 5%:
    return "持续放量"
elif slope < -5%:
    return "持续缩量"
elif 缩量后放量:
    return "缩量后放量"
else:
    return "量能平稳"
```

#### 相似度计算

```python
# 1. 均量比相似（允许±1.5倍差异）
ratio_diff = abs(cand_ratio - case_ratio)
sim = max(0, 1 - ratio_diff / 1.5)

# 2. 量能趋势一致性
if 同为缩量后放量:
    sim = 1.0
else:
    sim = 0.5

# 3. 最大量比相似（允许±3倍差异）
max_vol_diff = abs(cand_max - case_max)
sim = max(0, 1 - max_vol_diff / 3)
```

---

### 4. 价格形态相似度（25%）

**核心思想**：使用DTW（动态时间规整）算法比较归一化价格曲线的形状相似度。

#### 特征提取

| 特征 | 计算方法 | 含义 |
|------|----------|------|
| `normalized_curve` | (收盘价 - 最小值) / (最大值 - 最小值) | 归一化到0-1的价格曲线 |
| `max_drawdown` | (峰值 - 谷值) / 峰值 × 100% | 最大回撤百分比 |
| `breakout_strength` | 最后一日涨跌幅 | 突破力度 |
| `overall_trend` | 首尾比较 | 整体趋势方向 |

#### DTW算法

```python
# 归一化价格曲线
curve = (closes - min_price) / (max_price - min_price)

# 使用FastDTW计算曲线相似度
distance, _ = fastdtw(cand_curve, case_curve, dist=euclidean)
max_dist = max(len(cand_curve), len(case_curve))
curve_sim = max(0, 1 - distance / max_dist)
```

DTW算法的优势：
- 可以比较不同长度的序列
- 允许时间轴的弹性对齐
- 专注于形状相似度

#### 相似度计算

```python
# 1. 曲线形状相似度（DTW）
curve_sim = dtw_similarity(cand_curve, case_curve)

# 2. 回撤幅度相似（允许±15%差异）
drawdown_diff = abs(cand_drawdown - case_drawdown)
sim = max(0, 1 - drawdown_diff / 15)

# 3. 趋势方向一致性
if 同为上升或同为下降:
    sim = 1.0
else:
    sim = 0.5
```

## 窗口提取逻辑

**关键设计**：匹配时使用突破日期**之前**的数据（不包含突破当天）。

```python
# 取突破日期之前的数据（< breakout_date）
mask = df['date'] < breakout_date
window_df = df[mask].head(25)
```

**为什么？**

1. **捕捉整理形态**：突破前的25天通常是股票的整理/蓄势阶段
2. **避免突破干扰**：突破当天的大阳线会扭曲技术指标（价格等于或远超趋势线）
3. **比较"蓄势状态"**：我们希望找到同样处于蓄势待发的股票

## 相似度阈值说明

| 相似度 | 含义 | 建议操作 |
|--------|------|----------|
| ≥85% | 高度相似 | 重点关注，形态与成功案例非常接近 |
| 70-84% | 较为相似 | 值得关注，形态有较高相似性 |
| 60-69% | 一般相似 | 参考意义，形态有一定相似性 |
| <60% | 不太相似 | 可能不是目标形态 |

默认阈值：60%（低于此值不显示）

## 使用建议

### 1. 回看天数选择

```bash
# 短线整理（1-2周）
python3 main.py run --b1-match --lookback-days 15

# 中线整理（3-4周）- 默认
python3 main.py run --b1-match --lookback-days 25

# 长线整理（1-2月）
python3 main.py run --b1-match --lookback-days 40
```

### 2. 相似度阈值调整

```bash
# 严格匹配（只显示高度相似）
python3 main.py run --b1-match --min-similarity 75

# 宽松匹配（显示更多候选）
python3 main.py run --b1-match --min-similarity 50
```

### 3. 组合使用

```bash
# 回落碗中的股票 + B1匹配
python3 main.py run --b1-match --category bowl_center

# 靠近多空线的股票 + 高相似度
python3 main.py run --b1-match --category near_duokong --min-similarity 70
```

## 注意事项

1. **形态相似 ≠ 必然上涨**：相似度只是参考，不构成投资建议
2. **历史不代表未来**：案例都是历史成功案例，未来表现可能不同
3. **市场环境差异**：不同市场环境下的形态含义可能不同
4. **结合基本面**：技术分析应与基本面分析结合使用

## 扩展预留

B1完美图形匹配为后续扩展预留了空间：

- **B2完美图形**：可以添加第二套案例库（如跌破支撑后反弹的案例）
- **B3完美图形**：可以添加第三套案例库（如V型反转的案例）

每套案例库可以有不同的：
- 案例集合（不同的历史成功案例）
- 权重配置（不同的维度权重）
- 匹配逻辑（不同的相似度计算方式）

### 相似度计算流程

1. 提取候选股票25天特征向量
2. 与10个案例的特征向量逐一比较
3. 计算4个维度的相似度得分
4. 按权重加权求和得到总分
5. 按总分排序，返回最佳匹配
